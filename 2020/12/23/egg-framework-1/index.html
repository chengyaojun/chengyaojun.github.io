Egg框架开发：Symbol.for('egg#eggPath') is required on Application 错误

问题描述

使用Egg封装自己的的框架，在项目中npm link egg-framework-xxx测试，启动后报错：

环境信息

  • 操作系统： Win10
  • nodejs: v10.15.1
  • egg：2.15.1

根据以上的错误信息，定位到egg-ts-helper下的getLoader，

​在simple项目时，framework是egg，frameworkPath则是该项目中node_modules\egg。当时基于框架的项目时，framework是link的框架（framework-mandao）， frameworkPath则是link的框架的路径。

造成该问题的原因是，在link框架之后会有两个egg-core 。一个是框架的node_modules\egg-core，一个是项目的node_modules\egg-core

解决方案

因为framework、app中同时有egg-core。所以需要让两个模块使用一个egg-core即可。解决方式就是在项目中link框架node_modules\中的egg-core。使用shell脚本执行install、link操作
